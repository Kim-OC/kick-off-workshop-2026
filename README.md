# Ida Infront Kick-Off 2026 — AI Agent Workshop

**6-hour hands-on workshop: "Build Your AI Team"**

> *"Your idea is too good to stay in a notes app."* — Lovable

## Quick Facts

| | |
|---|---|
| **Duration** | 6 hours (09:00–15:00) |
| **Participants** | ~100 people, mixed roles (devs, PMs, finance, HR) |
| **Format** | 20 teams × 5 people, team leads with laptops |
| **Setting** | Physical kick-off event |
| **Coding required** | **ZERO** — everything via natural language |
| **Tools** | Lovable (app building) + KiloClaw (AI agents) |

## Workshop Goals (from Magnus)

1. **It should be fun** — competition, laughter, team spirit
2. **Meet people from other parts of the company** — cross-functional teams
3. **"Aha moment" about AI** — see agents *do* things, not just talk
4. **Take home ideas** — practical use cases for their own work

---

## Design Philosophy

**No code. All ideas.**

This workshop is designed so that *anyone* at Ida Infront can participate meaningfully — from a developer to an HR coordinator. The tools do the heavy lifting:

- **Lovable** — Describe what you want in plain English/Swedish. It builds the app.
- **KiloClaw (OpenClaw)** — Talk to an AI agent that has tools, memory, and autonomy.

The participants' job is not to code. It's to **think**, **decide**, and **validate**. The AI does the rest.

---

## Design Inspiration

This workshop draws from proven formats:

| Source | What we borrowed |
|---|---|
| **Microsoft Agent Build-A-Thon** | Teams of 2-5, hands-on labs, working demos, showcase session |
| **ASNT AI Agent Battle** | Competition format, 5-min team demos to judges, pre-event priming |
| **agents&me (Israel)** | Zero-code barrier, "by minute 45 you have a working agent" |
| **Lovable** | "AI cofounder" — describe, don't code. Real apps from natural language. |
| **Feet First Events** | Fun elements: team challenges, creative prompts, game-show energy |
| **Vistage research** | "Put them in a room, give them real data, require working demos" |

**Key design decision:** This is NOT a lecture. It's a **guided build-athon** where teams describe what they want in plain language and watch AI build it. Competition keeps energy high.

## Agenda

### 09:00–09:45 | Opening: "This Is Not a Chatbot" (45 min)

**Goal:** Blow people's minds. Set the tone.

**Flow:**
1. **Welcome (5 min)** — Workshop goals, team introductions
2. **Live demo: Kim in action (10 min)** — Magnus talks to Kim (his AI assistant) live on the big screen
   - Kim checks calendar, finds conflicts, drafts reschedule emails
   - Kim researches a topic and produces a structured summary
   - *Key message: "This agent has been running 24/7, with access to tools, files, and memory"*
3. **What is an AI agent? (10 min)** — Interactive, visual, no jargon
   - Chatbot vs Agent: **agents take action**
   - The Agent Loop: Ask → Think → Do → Show → Repeat
   - Real examples from Ida Infront
4. **Live demo: Lovable (5 min)** — "Watch me build an app in 60 seconds"
   - Describe: "Build a task tracker with a kanban board"
   - Watch the app appear on screen
   - *Key message: "If you can describe it, you can build it"*
5. **Team formation & icebreaker (15 min)** — Meet your team
   - Each team gets a name (AI-generated by the facilitator)
   - Quick round: "What's one task you wish you could delegate to AI?"

### 09:45–10:45 | Exercise 1: "Your First AI Agent" (60 min)

**Goal:** Everyone interacts with an AI agent. First "aha moment."

**Setup:** Team lead opens laptop. Team gathers around. KiloClaw instance is ready.

#### Part A: Guided Interaction (20 min)

The facilitator walks everyone through on the big screen. Teams follow along.

```
Prompt 1: "Introduce yourself. What can you help me with?"
Prompt 2: "Read the files in the workspace and tell me what this project does"
Prompt 3: "Find any issues or problems in this project"
```

Teams discuss: *Did the agent find things we expected? What surprised us?*

#### Part B: Team Mystery Investigation (25 min)

Each team gets a **mystery project** description loaded in their workspace. The project could be:
- A small web app they've never seen
- A document with company data
- A workflow description with hidden inefficiencies

**The team's job:**
1. Ask the agent to analyze the mystery
2. Ask follow-up questions based on what it finds
3. Debate: Do we agree with the agent's findings?
4. Ask the agent to suggest improvements

**Team roles (rotate):**
| Role | Who | What they do |
|---|---|---|
| Driver | Anyone | Types prompts into the terminal |
| Navigator | Anyone | Decides what to ask next |
| Domain Expert | PM/Finance/HR | Validates findings against real-world knowledge |
| Challenger | Anyone | Questions the agent's conclusions |
| Presenter | Anyone | Prepares to share findings |

*Everyone contributes. No coding skills needed.*

#### Part C: Group Share (15 min)

2-3 teams share what they discovered:
- "What surprised you?"
- "Did the agent miss anything obvious?"
- "What would you use this for at work?"

*☕ Break 10:45–11:00 (15 min)*

### 11:00–11:45 | Understanding the Platform (45 min)

**Goal:** Demystify how agents work. Connect the magic to engineering.

**Flow:**
1. **Architecture walkthrough (10 min)** — Visual, not technical:
   ```
   You → AI Agent → Tools (files, web, git, calendar) → AI Model
   ```
   - What's in the agent's "brain" (memory, skills, context)
   - What tools it has access to
   - Security & guardrails
2. **Live demo: Agent autonomy (10 min)**
   - "Research this topic, write a summary, save it, and commit it"
   - Watch the agent work through multiple steps on its own
3. **Lovable deep-dive (10 min)**
   - Show how Lovable turns descriptions into real apps
   - Show the code it generates (for the developers in the room)
   - Show how non-technical people can iterate: "Make the buttons blue, add a search bar, make it mobile-friendly"
4. **Interactive Q&A (15 min)**
   - "Can it send emails?" "Does it remember things?" "What if it makes a mistake?"
   - "How is this different from ChatGPT?"

### 11:45–12:00 | Introduce the Team Challenge (15 min)

**Goal:** Set up the afternoon build session. Build anticipation.

**Flow:**
1. **The Challenge briefing** — Announce the team build competition
2. **Two tools, infinite possibilities:**
   - **KiloClaw** — Research, analyze, plan, automate workflows
   - **Lovable** — Build actual apps from descriptions
   - Use one, use both, combine them
3. **Rules:**
   - Teams have 75 minutes to build something
   - Each team presents a 3-minute demo at 14:15
   - Judges score on: creativity, usefulness, teamwork, "wow factor"
4. **Team planning time** — Discuss what to build before the clock starts

*🍽️ Lunch 12:00–13:00 (60 min)*

### 13:00–14:15 | Exercise 2: Team Build Challenge (75 min)

**Goal:** Teams build something real with AI. Maximum "aha moments."

**No coding. Just describe what you want.**

#### Challenge Tracks (teams choose one):

#### Track A: "Build an App with Lovable" 🏗️
Describe an app in plain language. Lovable builds it. Iterate and improve.

**Starter ideas:**
- A team event planning tool
- A new employee onboarding checklist app
- A project dashboard showing status across teams
- A "who's who" directory with fun AI-generated descriptions
- A meeting agenda builder that pulls from calendars

**How it works:**
1. Team decides what to build (10 min)
2. One person drives Lovable on the laptop (the rest describe what they want)
3. Describe features in plain language: "Add a kanban board" → app updates
4. Iterate: "Make it look more professional" → redesign
5. Test it as a team
6. Prepare a 3-minute demo

#### Track B: "Design a Workflow Agent" 🤖
Design an AI agent that automates a real work process. Use KiloClaw to prototype.

**Starter ideas:**
- **Meeting Prep Agent** — Reads calendar, gathers docs, creates agenda
- **Expense Report Assistant** — Reads receipts, categorizes, fills forms
- **Onboarding Buddy** — Answers new hire questions from company docs
- **Weekly Report Generator** — Pulls data, writes the status report
- **Customer Meeting Prep** — Researches customer, creates briefing doc

**How it works:**
1. Pick a workflow (or invent one) (10 min)
2. Describe the workflow to the agent in detail (10 min)
3. Ask the agent to design the automation: inputs, steps, outputs (15 min)
4. Create the prompts/templates the agent would use (15 min)
5. Test it with sample data (10 min)
6. Prepare a 3-minute demo (15 min)

#### Track C: "AI Solves Our Problem" 💡
Identify a real problem and use AI to research, design, and pitch a solution.

**Starter ideas:**
- "New hires take 3 months to become productive" → AI onboarding solution
- "We spend 4 hours/week on status reports" → automated reporting
- "Customer onboarding has a 30% drop-off rate" → AI-guided onboarding
- "Too many meetings" → AI meeting optimizer

**How it works:**
1. Brainstorm problems as a team (10 min)
2. Pick the most interesting one (5 min)
3. Research with the agent: how have others solved this? (20 min)
4. Design the AI-powered solution concept (15 min)
5. Build a simple prototype in Lovable OR prepare a compelling pitch (15 min)
6. Prepare a 3-minute demo (10 min)

#### During the Build

- **Coaches circulate** — 3-4 facilitators walk around helping teams
- **Time check-ins** — At 30 min and 60 min
- **The non-laptop people are the most valuable:**
  - They know the real problems
  - They validate whether solutions make sense
  - They prepare the demo presentation
- **Everyone participates** — not just the person typing

### 14:15–14:45 | Team Demos & Judging (30 min)

**Goal:** Show off work. Celebrate. Learn from each other.

**Flow:**
1. **Each team gets 3 minutes** — Demo what they built
2. **Judges score** on 4 criteria (1-5 points each):
   - 🎨 **Creativity** — Did they think outside the box?
   - 💼 **Usefulness** — Could this actually help someone at work?
   - 🤝 **Teamwork** — Did everyone contribute?
   - ✨ **Wow factor** — Did it make people go "whoa"?
3. **Prize categories:**
   - 🏆 **Best Overall** — Highest total score
   - 😂 **Most Fun** — Made everyone laugh
   - 💡 **Most Useful** — Could be shipped tomorrow
   - 🌟 **Best Cross-Team Collaboration** — HR + devs + finance working together
   - 🚀 **Best "I Didn't Know AI Could Do That" Moment**

### 14:45–15:00 | Wrap-Up & Next Steps (15 min)

**Goal:** Cement learnings. Point to Monday morning.

**Flow:**
1. **What we learned** — Key takeaways
2. **Your AI journey starts Monday:**
   - How to get your own KiloClaw instance
   - How to access Lovable (if available company-wide)
3. **Feedback** — Quick survey (QR code on screen)
4. **Group photo** — With all the teams and their creations

## Technical Setup

### KiloClaw Instances (20 total)

| Component | Details |
|---|---|
| **Platform** | KiloClaw (OpenClaw-based) on Fly.io |
| **Model** | Claude (balanced tier) |
| **Access** | Web URL + session token (pre-configured) |
| **Channels** | CLI only for workshop |
| **Workspace** | Pre-loaded with mystery projects and challenge materials |

### Lovable Access

| Option | Pros | Cons |
|---|---|---|
| **A: Company subscription** | Real apps, everyone gets access afterward | Cost, IT setup time |
| **B: Shared team accounts** | Quick to set up, 20 accounts max | Shared usage, limited features |
| **C: Demo-only** | No setup needed | Teams can't build live apps |

**Recommendation:** Option B for the workshop, then evaluate Option A company-wide based on engagement.

### Instance Provisioning

```
20 teams × 1 laptop per team
├── Laptop runs browser (Lovable) + terminal (KiloClaw)
├── KiloClaw instance:
│   ├── Pre-configured agent profile (workshop SOUL.md)
│   ├── Mystery project for Exercise 1
│   ├── Quick reference card
│   └── Challenge instructions
└── Lovable account (Option A or B):
    ├── Blank project ready to build
    └── No setup needed — just log in and start describing
```

### Why No Personal Accounts Needed

- KiloClaw instances are pre-provisioned with tokens
- Lovable accounts are pre-created for teams
- **Zero friction** — team leads just open a browser and start

### WiFi Requirements

- **20 concurrent browser sessions** (Lovable) + **20 CLI sessions** (KiloClaw)
- Lovable needs moderate bandwidth (real-time UI updates)
- KiloClaw is lightweight (text-based)
- **Backup:** 2 mobile hotspots for emergency access

## Exercise 1: Mystery Project Materials

### Mystery 1: "The Bug Hunt"

A description of a small application with intentional problems:

> *"This is a customer feedback form app. Customers fill in their experience rating (1-5 stars), leave a comment, and submit. The data goes to a spreadsheet. There are some issues..."*

**Hidden issues for the agent to find:**
- No validation — empty submissions accepted
- No duplicate detection — same person can submit 100 times
- Star rating is text, not number — can't calculate average
- No confirmation after submission

### Mystery 2: "The Workflow Detective"

A description of a real company workflow with hidden inefficiencies:

> *"Every month, the finance team collects expense reports from 50 employees. They check each receipt, categorize it, enter it into the system, flag anomalies, and produce a summary report. The process takes 3 days."*

**Hidden inefficiencies for the agent to find:**
- Receipt photos could be processed by AI (OCR + categorization)
- Anomaly detection could be automated (rules-based)
- Summary report could be generated automatically
- Employee submission could be a simple app

### Mystery 3: "The Data Puzzle"

A set of company data with patterns to discover:

> *"Here are support ticket numbers for the last 6 months, categorized by type. Find patterns and suggest improvements."*

**Patterns for the agent to find:**
- Certain ticket types spike after product releases
- Resolution time correlates with ticket category
- Self-service could reduce certain categories by 40%

**Magnus + Kim live on screen:**

2. Kim responds with schedule, highlights conflicts
